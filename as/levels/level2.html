2.1 Control Structures

  • What are Control Structures? Control structures dictate the flow of a program. They enable decisions to be made based on conditions, allowing different parts of the algorithm to run depending on those conditions.
Example A - Simple decision flow
DECLARE flag : BOOLEAN
INPUT flag
IF flag = TRUE THEN
OUTPUT "Condition is true."
ELSE
OUTPUT "Condition is false."
ENDIF

Use clear indentation to show which lines belong to each branch. End each IF with ENDIF.

2.2 IF Statements

  • IF-THEN-ELSE: Run one block when a condition is true; otherwise run the alternative block after ELSE.
Example A - Basic IF-THEN-ELSE
DECLARE age : INTEGER
INPUT age
IF age >= 18 THEN
OUTPUT "You are an adult."
ELSE
OUTPUT "You are not an adult."
ENDIF
  • Nested IF: Put an IF inside another IF to handle multiple conditions step by step.
Example B - Nested IF
DECLARE number : INTEGER
INPUT number
IF number > 0 THEN
OUTPUT "The number is positive."
ELSE
IF number < 0 THEN
OUTPUT "The number is negative."
ELSE
OUTPUT "The number is zero."
ENDIF
ENDIF

When nesting, align each ENDIF with its matching IF so the structure stays clear.

2.3 Relational and Logical Operators

  • Relational operators: compare two values and produce TRUE or FALSE.
Example A - Relational operators list
> // greater than
< // less than
>= // greater than or equal to
<= // less than or equal to
= // equal to
<> // not equal to
  • Logical operators: combine conditions.
Example B - Using logical operators
DECLARE age : INTEGER
DECLARE hasLicense : BOOLEAN
INPUT age
INPUT hasLicense
IF age >= 18 AND hasLicense THEN
OUTPUT "You are eligible to drive."
ELSE
OUTPUT "You are not eligible to drive."
ENDIF

AND needs all conditions TRUE. OR needs at least one TRUE. NOT flips TRUE to FALSE and vice versa.

2.4 CASE Statements

  • CASE: choose one action from several options based on a single expression.
Example A - CASE on a grade
DECLARE grade : CHAR
INPUT grade
CASE OF grade
'A' : OUTPUT "Excellent"
'B' : OUTPUT "Good"
'C' : OUTPUT "Fair"
'D' : OUTPUT "Poor"
OTHERWISE : OUTPUT "Fail"
ENDCASE

In a CASE, only the matching branch runs. Use OTHERWISE to handle any value that does not match a listed case.
