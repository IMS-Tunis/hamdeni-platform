5.3a Types of Cyber Security Threats

  • A brute-force attack is when a hacker tries to guess a user's password by testing many combinations until the correct one is found.
    • This can be done manually or automatically using a program that tries hundreds or thousands of guesses per second.
    • Hackers often begin with commonly used passwords like ‚Äú123456‚Äù or ‚Äúpassword‚Äù before trying random combinations.
    • The longer and more complex a password is, the harder it becomes to guess.
    • The goal of a brute-force attack is to steal personal data or use someone else's account to buy things without permission.
  • Data interception means capturing data as it travels between devices across a network, often without the user's knowledge.
    • A common tool used is a packet sniffer, which inspects packets of data sent through routers or Wi-Fi.
    • In wired networks, data packets can be intercepted using software installed at a connection point.
    • In wireless networks, attackers may use a technique called wardriving, using a laptop and antenna to collect signals from outside buildings.
    • The aim is often to steal confidential information for identity theft or fraud.
  • A Distributed Denial of Service (DDoS) attack floods a network, server, or service with an overwhelming amount of fake traffic.
    • This makes the system unable to handle real user requests, causing it to slow down or crash.
    • A DDoS attack is launched using many computers (a botnet) infected with malware to send repeated requests.
    • The malware allows hackers to control those devices remotely and coordinate the attack.
    • It can stop users from checking email, accessing websites, or using online banking.
  • Hacking is gaining unauthorized access to a system or network, usually with the intent to steal, corrupt, or leak data.
    • Hackers look for system weaknesses such as unpatched software, open ports, or weak login credentials.
    • Once inside, they may delete files, access private data, or cause system failure.
    • Even encrypted data can be deleted or modified if hacking is successful.
  • Malware refers to malicious software designed to harm, steal, or interrupt the normal operation of a system.
    • Viruses replicate themselves and may delete files or consume storage, making the computer crash.
    • Worms spread across networks without needing user interaction, often slowing down or damaging systems.
    • Trojan horses pretend to be legitimate software but contain harmful code that activates when the user installs it.
    • Spyware secretly monitors user activity and sends information (like keystrokes) to a hacker.
    • Adware shows unwanted ads and may redirect your browser without permission.
    • Ransomware encrypts your files and demands payment to unlock them.
    • Rootkit is malware that secretly allows continued access to a system while hiding its presence.
  • Phishing is a technique where users are tricked into clicking fake links or giving away personal data.
    • Although phishing is a cyber threat, it is not classified as a type of malware.
    • The attacker sends an email that looks like it‚Äôs from a real bank or service.
    • The email urges the user to click a link or update account details on a fake website.
    • Signs of phishing include generic greetings like "Dear Customer", spelling errors, or mismatched URLs.
  • Pharming redirects the user to a fake website even if they typed the correct URL into their browser.
    • This can be caused by malicious code on the user‚Äôs computer or tampering with DNS records.
    • One common method is DNS cache poisoning, which changes the real IP address to that of the fake site.
    • The fake website collects usernames, passwords, or banking details.
    • Unlike phishing, pharming doesn‚Äôt require the user to click a link‚Äîit happens automatically.
  • Social engineering uses deception to trick people into giving away private data or breaking security rules.
    • It manipulates emotions like fear, curiosity, or trust.
    • Common tactics include scareware, fake phone calls from ‚Äúsupport staff,‚Äù and baiting with USB drives.
    • The attacker often doesn‚Äôt need technical skills‚Äîonly the ability to convince the victim.

5.3b Preventing Cyber Attacks and Threats

  • Use a strong password that is long, includes upper- and lower-case letters, numbers, and symbols.
    • Avoid passwords based on birthdays, pet names, or common words.
    • Strong passwords are harder for brute-force attacks to guess.
  • Limit the number of times a user can enter the wrong password before being locked out.
    • This makes brute-force attacks less likely to succeed by limiting guessing attempts.
    • Another helpful method is CAPTCHA, which stops automated programs from submitting login attempts.
  • Use authentication methods to verify user identity securely.
    • Authentication may use something the user knows (e.g. password), has (e.g. mobile phone), or is (e.g. fingerprint).
  • Use two-step verification for added security.
    • After entering a password, a code is sent to a trusted device.
    • This protects against hacking and brute-force attacks by requiring two separate pieces of information.
  • Use biometric methods like fingerprint or retina scans for secure, user-specific access.
    • These are difficult to fake and always unique to the person.
    • They add an extra layer of protection against unauthorized access.
  • Assign different access levels to users in a system based on what data they need.
    • This ensures that only authorized people can view or modify sensitive information.
    • Helps limit damage in case of social engineering or hacking.
  • Apply privacy settings in social media and browsers.
    • This prevents unknown users from seeing your information or tracking your activity.
    • Useful in blocking social engineering attempts.
  • Install and regularly update anti-malware software.
    • This detects, quarantines, and removes threats like viruses, worms, spyware, and ransomware.
    • Anti-spyware software can block programs that try to steal information or hijack your webcam.
  • Keep software up to date by enabling automatic updates.
    • Updates fix security holes that hackers might exploit.
    • Delaying updates increases the risk of malware infections.
  • Use a firewall to filter incoming and outgoing traffic.
    • Firewalls can help block various malware such as viruses, worms, trojan horses, spyware, ransomware, and adware.
    • Firewalls block unwanted data, warn about suspicious access attempts, and enforce security rules.
    • They are essential for defending against malware, phishing, pharming, and hacking.
  • Use a proxy server as an extra layer between user and internet.
    • Proxy servers filter web traffic, cache pages, hide user IP addresses, and help prevent DDoS attacks.
    • They stop malicious traffic before it reaches a real server.
    • Proxy servers can also hide your IP address, cache frequently visited content, and block harmful IP addresses.
  • Use SSL (Secure Sockets Layer) to encrypt data transmitted online.
    • Encryption scrambles data so that only the intended recipient can understand it.
    • Websites with ‚ÄúHTTPS‚Äù and a padlock symbol use SSL.
  • Always check emails carefully before clicking links or entering data.
    • Look for spelling errors, urgent language, or generic greetings (e.g., ‚ÄúDear customer‚Äù). These are signs of phishing.
    • Hover over links to verify the true destination.
  • Never click on suspicious pop-ups or fake software warnings.
    • These might lead to phishing or download malware disguised as antivirus tools.
  • Be cautious about where you download software from.
    • Only use trusted sources to avoid hidden malware or trojan horses.
    • Helps protect against phishing, pharming, and social engineering.
  • Create regular backups of your important data on a separate device or cloud service.
    • If your computer is attacked by ransomware, you can restore files without paying a ransom.
