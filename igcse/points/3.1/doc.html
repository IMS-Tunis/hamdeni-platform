3.1a: Role of the CPU and Microprocessor

  • The central processing unit (CPU), also called a microprocessor or processor, is the main part of the computer that processes all data and instructions, similar to the brain in a human body.
    • It is found in all modern devices like PCs, tablets, and smartphones, often built into a single integrated circuit.
    • The CPU usually contains three key parts: the control unit (CU), the arithmetic and logic unit (ALU), and registers connected through buses.
  • In smaller electronic systems, like washing machines or traffic lights, a similar component called a microprocessor is used.
    • These microprocessors are part of embedded systems, which are designed to carry out one specific task rather than multiple ones.
  • A washing machine uses a microprocessor to control dedicated tasks like selecting and executing wash cycles.
  • If your brain had a CPU, it would be like your personal assistant - sorting tasks, solving math, and making sure you don‚Äôt forget lunch. A washing machine‚Äôs microprocessor is like a helper that just knows how to do laundry - nothing else!
  • Example: In a parking system, a microprocessor checks if a sensor is activated, compares how long a car has been parked against the time paid, and sends a signal to an output device (like a barrier) if the time is exceeded.

3.1b: Von Neumann Architecture, CPU Components, FDE Cycle

  • The Von Neumann architecture is a design model where both data and programs are stored in memory and processed by a central CPU using a repeated Fetch‚ÄìDecode‚ÄìExecute (FDE) cycle.
    • The architecture uses a system of components including: Program Counter (PC), Memory Address Register (MAR), Memory Data Register (MDR), Current Instruction Register (CIR), Arithmetic Logic Unit (ALU), Accumulator (ACC), Control Unit (CU), and the system clock.
    • The CPU connects to memory and devices using three main system buses: - Address bus: sends memory locations in one direction (unidirectional) - Data bus: sends and receives data (bidirectional) - Control bus: sends control signals (usually bidirectional)
  • The Fetch stage involves several steps: - The PC stores the address of the next instruction and sends it to the MAR using the address bus. - The instruction at that address is copied from immediate access store (IAS) into the MDR. - The MDR passes the instruction to the CIR, where it is ready to be decoded. - The PC is incremented by 1 to point to the next instruction.
  • In the Decode stage, the CIR uses the instruction set to decode the instruction so the CPU understands what to do.
  • In the Execute stage, the CU sends control signals to carry out the decoded instruction. If calculations are needed, the ALU performs them using the ACC.
  • Reading from RAM: MAR holds address 1111 0001 ‚Üí RAM sends data 0101 1011 to MDR. Why it matters: Shows how CPUs find and process instructions step-by-step.
  • Think of the fetch-decode-execute cycle like ordering pizza: You look up the pizza (fetch), read the recipe (decode), and cook it (execute). The CPU does this millions of times per second - but sadly, no pizza at the end.

üîπ 3.1c: Core, Cache, and Clock - Factors Affecting Performance

  • CPU performance is about how quickly the CPU can carry out instructions, and it's affected by three main things: clock speed, number of cores, and size of cache.
  • The clock speed is measured in gigahertz (GHz) and tells how many clock cycles can be completed per second. A faster clock can increase processing speed but may also cause overheating or errors if pushed beyond safe limits (overclocking).
  • Cores are individual processors within the CPU. More cores allow more instructions to be processed at the same time.
    • A dual-core CPU can process two instructions at once; a quad-core can do four, allowing multitasking.
    • However, more cores don‚Äôt always double performance because the CPU must coordinate communication between cores.
  • Cache is high-speed memory inside the CPU that stores frequently used data and instructions.
    • Using cache is faster than accessing RAM, so more cache generally means better performance-up to a point.
  • Example: Increasing clock speed from 2.4 GHz to 3.5 GHz improves CPU performance by allowing more instructions per second to be executed, speeding up processing and improving responsiveness.
  • A 2.4 GHz dual-core CPU processes 4.8 billion instructions/second: Calculation: 2.4 billion cycles/core √ó 2 cores = 4.8 billion. Why it matters: Shows how cores and speed multiply performance.
  • More cores in a CPU are like more chefs in a kitchen - they can cook faster, but only if they don‚Äôt bump into each other. And cache is like a spice rack on the counter - faster than running to the pantry (RAM)!

üîπ 3.1d: Instruction Set

  • An instruction set is a list of all low-level machine code commands a CPU can understand and execute.
    • Each instruction consists of an opcode (the action to be performed) and an operand (the data or location the action applies to).
    • Software written in high-level programming languages is translated into these low-level instructions by compilers or interpreters.
  • The X86 instruction set (used in Intel CPUs) allows different computers to run the same software.
  • An instruction set is like a language your CPU speaks. If you shout ‚ÄúWash the dishes!‚Äù at your toaster, it won‚Äôt respond - wrong instruction set. CPUs need the right opcodes to do the job!

üîπ 3.1e: Embedded Systems

  • An embedded system is a computer system built into a larger device to perform one specific, dedicated function.
    • Most embedded systems are not easy to reprogram and are designed to carry out their specific tasks reliably without frequent updates.
    • Embedded systems may include microcontrollers (with CPU, RAM, ROM on a chip), microprocessors (CPU only), or SoC (System on Chip) that combines everything.
  • Embedded systems are used in devices like washing machines, GPS units, security alarms, vending machines, lighting systems, and medical equipment.
    • They may take input from sensors (e.g., temperature, pressure), process it, and give outputs such as displaying information or controlling motors.
  • Some embedded systems are programmable (software can be updated), while others are fixed and need replacement if outdated.
  • Embedded systems are used in devices like washing machines, GPS units, security alarms, vending machines, lighting systems, and medical equipment.
  • A car‚Äôs ABS braking system: Inputs: Sensors detecting wheel speed. Output: Adjusts brake pressure instantly.
  • Example: A vending machine uses sensors to read coin values and temperatures and actuators to release snacks. All of this is controlled by an embedded system, making the machine work automatically.
  • Example: A smartwatch is not a general-purpose computer because it is built for limited, specific functions like fitness tracking or notifications and cannot run a wide range of software like a laptop or desktop computer.
  • An embedded system is like that friend who only talks about one thing - cars, games, or cake. They‚Äôre great at that one thing, but don‚Äôt expect them to help with your math homework.
