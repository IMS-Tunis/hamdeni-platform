1.2a: Understand how and why a computer represents text and the use of character sets, including ASCII and Unicode

  • Computers store and process text as binary. A character set is a defined list of characters where each character has a fixed binary code.
    • ASCII (American Standard Code for Information Interchange) is a character set that assigns a unique code to common characters.
    • Unicode (standard character set) supports a much wider range of characters and languages than ASCII.
  • ASCII case codes: Hex means hexadecimal notation used in character set tables. The letter A has the hexadecimal code 41 and the letter a has the hexadecimal code 61. This shows a consistent mapping between characters and their stored codes.
  • ASCII in different number bases: The letter Q has denary code 81, and the letter N has the binary code 01001110. This illustrates that the same character mapping can be expressed in denary or in binary.
  • Decoding a short ASCII binary message: Split the bit stream into 8-bit chunks, convert each chunk to its character using the ASCII table, and join them. For example, 01000011 01101111 01101101 [â€¦] 00100001. This produces a readable text message.
  • Character sets work like a bilingual dictionary: each entry on the left always points to one exact item on the right. ASCII and Unicode are different dictionaries, with Unicode having many more entries.

1.2b: Understand how and why a computer represents sound, including the effects of the sample rate and sample resolution

  • Sound is recorded by taking samples of the sound wave at set time intervals. Each sample is stored as a binary value so the computer can save and play back the sound.
    • Sampling uses an analogue to digital converter to measure the sound wave amplitude at regular time intervals, and approximate amplitude values are stored in binary.
    • The sample rate is the number of samples taken each second, measured in hertz. Hertz means per second (1 Hz equals 1 sample per second). Increasing the sample rate captures the sound more accurately and increases the file size.
    • The sample resolution is the number of bits used to store each sample. Using more bits allows more possible sample values and increases the file size.
  • An audio CD uses a sample rate of 44.1 kHz and a 16-bit resolution. This means 44 100 samples are taken each second and each sample is stored using 16 bits, which produces a high quality recording and a larger file size.
  • Think of two tools: a fast stopwatch and a detailed ruler. The stopwatch speed is the sample rate; the ruler detail is the bit depth. Faster checks and finer measurements make the copy closer to the original sound, and the file size increases.

1.2c: Understand how and why a computer represents an image, including the effects of the resolution and colour depth

  • A bitmap image is made of pixel. A pixel is the smallest image element of a single colour. In a black and white image, each pixel can be stored with a single bit that represents black or white.
    • Image files store metadata such as width and height that helps software reconstruct the image correctly when it is displayed.
    • Image resolution is the number of pixels in the image, given by its width multiplied by its height. Here, resolution refers to the total number of pixels (width Ã— height). Higher resolution shows more detail and increases the file size.
    • colour depth is the number of bits used for each pixel. More bits allow more possible colours and increase the file size.
    • Increasing colour depth allows a greater range of colours and more detail, producing images that are closer to the original content.
  • Pixel count from resolution: An image that is 4096 pixels wide and 3072 pixels high contains 4 096 Ã— 3 072 = 12 582 912 pixels. This shows how resolution directly affects how much data must be stored.
  • Reconstructing a small black and white image from bits: If an image is 9 pixels wide and 10 pixels high, and we are given a bit string where 1 means black and 0 means white, the computer fills pixels row by row using the width and height. This shows that knowing the dimensions lets the computer turn stored bits back into a visible image.
