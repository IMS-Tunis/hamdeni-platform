1.3 Data storage and compression

Computer Science IGCSE notes, prepared by Dr. Hamdeni

1.3a: Understand how data storage is measured

  • Data size is measured with basic units that build on each other.
    • A bit is the smallest unit of data and has a value of 0 or 1.
    • A nibble is 4 bits.
    • A byte is 8 bits.
  • Larger units use 1024 multiples and follow a fixed pattern.
    • 1 kibibyte (KiB) is 1024 bytes.
    • 1 mebibyte (MiB) is 1024 KiB.
    • 1 gibibyte (GiB) is 1024 MiB.
    • 1 tebibyte (TiB) is 1024 GiB.
    • 1 pebibyte (PiB) is 1024 TiB.
    • 1 exbibyte (EiB) is 1024 PiB.
  • Converting GiB to bytes
    Start value: 64 GiB.
    Step 1 (GiB → MiB): 64 × 1024 = 65 536 MiB.
    Step 2 (MiB → KiB): 65 536 × 1024 = 67 108 864 KiB.
    Step 3 (KiB → bytes): 67 108 864 × 1024 = 68 719 476 736 bytes.
    This shows how multiplying by 1024 at each step converts to the next smaller unit until we reach bytes.
  • Converting between units
    Case A (KiB → MiB): 512 KiB ÷ 1024 = 0.5 MiB (divide by 1024 to move up one unit).
    Case B (GiB → MiB): 4 GiB × 1024 = 4096 MiB (multiply by 1024 to move down one unit).
    Case C (KiB → bits): 1 KiB = 1024 bytes and 1 byte = 8 bits, so 1024 × 8 = 8192 bits.
    These examples show how to convert between adjacent storage units and between bytes and bits.

1.3b: Calculate the file size of an image file and a sound file

  • Image file size can be calculated from resolution and colour depth, and multiplied by the number of images when there are several.
    • Resolution is width × height in pixels; multiplying width by height gives the total number of pixels.
  • Ten small images (step-by-step)
    Given: 10 images, each 100 × 150 pixels, with 8-bit colour depth.
    Step 1 (pixels per image): 100 × 150 = 15 000 pixels.
    Step 2 (bits per image): 15 000 × 8 = 120 000 bits (8 bits per pixel).
    Step 3 (bits for all images): 120 000 × 10 = 1 200 000 bits (multiply by the number of images).
    Step 4 (bytes): 1 200 000 ÷ 8 = 150 000 bytes (8 bits = 1 byte).
    Step 5 (KiB): 150 000 ÷ 1024 ≈ 146.5 KiB (divide by 1024 to convert from bytes to KiB).
    This sequence explains each calculation and why each conversion is used.
  • Single image size to MiB (step-by-step)
    Given: 1 image, 2 048 × 2 048 pixels, 16-bit colour depth.
    Step 1 (pixels): 2 048 × 2 048 = 4 194 304 pixels.
    Step 2 (bits): 4 194 304 × 16 = 67 108 864 bits.
    Step 3 (bytes): 67 108 864 ÷ 8 = 8 388 608 bytes.
    Step 4 (MiB): 8 388 608 ÷ (1024 × 1024) = 8 MiB.
    Each step shows the conversion from pixels to bits, then to bytes, and finally to MiB.
  • Sound file size can be calculated from the sample rate, the sample resolution, and the length of the track.
    • Use: size in bits = sample rate × sample resolution × length in seconds.
    • For stereo audio, multiply the mono result by 2.
    • Sample rate is given in hertz Hz and sample resolution is the number of bits stored for each sample.
    • Higher sample resolution stores more detail per sample and increases file size.
  • One hour of audio using the stereo rule (step-by-step)
    Given: length = 60 minutes, sample rate = 44 100 Hz, sample resolution = 16 bits, stereo audio.
    Step 1 (seconds): 60 minutes × 60 = 3600 seconds.
    Step 2 (mono bits): 44 100 × 16 × 3 600 = 2 540 160 000 bits.
    Step 3 (stereo bits): 2 540 160 000 × 2 = 5 080 320 000 bits (two channels).
    Step 4 (bytes): 5 080 320 000 ÷ 8 = 635 040 000 bytes.
    Step 5 (MiB): 635 040 000 ÷ 1 048 576 ≈ 605 MiB.
    This shows how sample rate, resolution, length, and channel count determine file size.
  • Unit conversions used in these calculations must be applied carefully.
    • Convert bits to bytes by dividing by 8.
    • Convert bytes to KiB by dividing by 1024.
    • Convert bytes to MiB by dividing by 1024 twice.

1.3c: Understand the purpose of and need for data compression

  • Compression reduces the size of a file and makes it easier to handle.
    • Smaller files need less storage space.
    • Smaller files take less time to transmit, so uploads and downloads are quicker.
    • Smaller files need less network bandwidth, so data transfer is faster.
    • Network bandwidth is the maximum data transfer rate across a network and is measured in bits per second.
    • Smaller files can reduce storage costs and can increase the number of files that can be attached to one email.
  • Compression is like vacuum-packing a suitcase: the same clothes take less space, so the bag is easier to carry and moves through the airport faster, just like smaller files travel quicker on a network.

1.3d: Understand lossy and lossless compression methods

  • Lossy compression reduces file size by permanently removing data judged unnecessary for typical use.
    • For images, lossy methods can reduce colour depth or resolution to store fewer bits per pixel.
    • For sound, lossy methods can remove sounds that are outside normal hearing or softer sounds that occur with louder ones.
    • Perceptual music shaping removes sounds that are not normally heard by the human ear, so the file becomes smaller while remaining acceptable to listeners.
    • MP3 is a lossy audio format that uses perceptual music shaping.
  • Text compression using a lookup table
    Message: WHEN IT IS SNOWING HEAVILY LOOK OUTSIDE. LOOK OUTSIDE IT IS SNOWING HEAVILY.
    Step 1 (uncompressed size): 62 characters → 62 bytes (1 byte per character).
    Step 2 (build the table): store repeated words once and record their positions.
    Step 3 (compressed size): store references to repeated words → 46 bytes.
    Step 4 (saving): 62 bytes - 46 bytes = 16 bytes saved (about 26 percent).
    This shows how repeated items reduce storage when references are used instead of storing the same text many times.
  • Run-length encoding RLE groups consecutive repeated data and stores the number of repeats together with the data value.
  • RLE is effective when long runs of identical items occur.
  • What lossy audio removes
    A lossy encoder can remove very high-frequency components outside the hearing range and very quiet components masked by louder sounds, so the file becomes much smaller while remaining acceptable to most listeners.
    This explains the idea behind perceptual removal in audio compression.
  • MP3 size reduction
    MP3 uses perceptual music shaping and can reduce the size of a normal music file by about 90 percent while keeping quality acceptable for most listening.
  • When to choose lossless
    Choose lossless when losing any data would be unacceptable, for example a spreadsheet that must be restored exactly.
  • Lossy versus lossless is like making a smoothie versus freezing whole fruit: the smoothie removes peels and seeds and you cannot get them back, while freezing keeps everything so you can restore it exactly later.
