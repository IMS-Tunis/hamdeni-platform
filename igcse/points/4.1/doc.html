4.1 Hardware and Software

Computer Science IGCSE notes, prepared by Dr. Hamdeni

4.1a

  • Software is a set of instructions that tells a computer what to do and allows the user to do useful tasks on devices such as PCs, laptops, tablets, or mobile phones.
  • Without software, the hardware itself cannot do anything meaningful.
  • There are two main types of software: system software and application software.
    • System software manages the running of hardware and other software and provides a platform on which other software can run. It ensures that hardware and software work properly together and maintains efficiency.
    • Application software allows users to perform specific tasks like writing letters or analyzing data. It can be a single program or a suite.
  • Typical examples of system software include:
    • Device drivers enable peripherals to communicate with the operating system.
    • Utility programs carry out maintenance tasks such as virus checking, backups, disk repair, file compression, and security management.
    • The operating system manages basic functions like input/output, error handling, running programs, and security.
  • Typical examples of application software include:
    • Word processors create and edit text documents, allowing tools like formatting, spell checking, and importing images.
    • Spreadsheets organize numerical data into rows and columns, performing calculations using formulas and providing graphs.
    • Databases store and analyze data using tables, allowing users to query, modify, and report on the data.
    • Web browsers retrieve, display, and navigate web pages.
  • Defragmentation: A defragmenter rearranges fragmented file pieces so that they are stored contiguously on a disk. This reduces the movement of the disk's read-write head, which improves file access speed and overall computer performance.
  • Example: Using a Printer ‚Äî When you plug in a new printer, the device driver (system software) ensures the OS and printer can communicate, letting applications print.
  • Common applications like word processors or web browsers can run on various devices such as PCs, tablets, or smartphones, performing the same basic tasks across different platforms.
  • Think of system software as the ‚Äúinvisible stage crew‚Äù of a play. They handle lights, curtains, and props (hardware and drivers) so the actors (applications) can perform for the audience (you) without any chaos. Without the stage crew, the actors would be standing in the dark, confused!

4.1b

  • An operating system (OS) is system software that lets users interact with hardware and manages computer operations. Examples include Windows, Mac OS, Android, and iOS.
  • The OS provides a user interface allowing data input and output. There are different types of interfaces:
    • A Graphical User Interface (GUI) uses visual elements like windows, icons, menus, and pointers, making it easy and intuitive for beginners.
    • A Command Line Interface (CLI) requires users to type exact text commands, suitable for advanced users needing precise system control.
    • A Natural Language Interface allows users to type or speak normally, and the OS interprets these commands without requiring exact syntax.
  • The operating system manages files, including creating, opening, deleting, renaming, copying, moving, and organizing files and folders.
  • The OS manages peripherals through device drivers, software that translates data between the hardware and OS. Without drivers, peripherals cannot function properly.
  • The OS manages memory by controlling data movement between RAM (Random Access Memory, a type of temporary storage) and permanent storage, ensuring applications have enough memory, and preventing memory conflicts.
    • Managing memory involves allocating memory to different processes, preventing processes from accessing each other‚Äôs memory, and freeing up memory when processes complete.
  • The OS manages multitasking by allowing multiple applications to run simultaneously, allocating resources efficiently, and switching tasks as needed.
  • The OS manages system security, user accounts, and access rights. It applies updates, manages antivirus software, firewalls, user permissions, and data protection.
    • The OS also helps recover data if files become lost or corrupted, thanks to built-in backup and restore tools.
  • The OS provides a platform for running applications by loading applications into memory and managing their execution.
  • Voice assistants like Amazon's Alexa or Apple's Siri allow users to speak normal commands, interpreting the spoken words to carry out tasks without requiring technical commands or syntax knowledge.
  • Memory conflict prevention: The OS stops a game and a word processor from overwriting each other‚Äôs data in RAM.
  • Imagine your OS as a super-busy party host. It takes coats (stores files), introduces guests (connects devices), stops arguments (memory conflicts), and ensures everyone gets a slice of cake (CPU time). And when the music stops suddenly, that‚Äôs an ‚Äúinterrupt‚Äù telling the host to fix the speaker!

4.1c

  • Applications run on the operating system; the operating system runs on firmware; firmware is loaded through a bootstrap process when the computer starts.
  • Firmware is software stored permanently in Read-Only Memory (ROM), a type of memory whose contents cannot be easily altered. Firmware is executed when the computer starts to locate and load the OS.
    • Firmware stores instructions to start the computer (boot up), provides the operating system with a platform to run on, and manages communication with hardware.
  • The BIOS (Basic Input/Output System) is firmware stored in EEPROM (Electrically Erasable Programmable Read-Only Memory, a type of memory that can be reprogrammed with electrical charge). BIOS provides low-level control for hardware devices and loads the OS when the computer starts.
  • The bootstrap program is the first program executed when starting a computer, checking hardware components, and loading firmware.
    • On phones and tablets, the OS is stored in solid-state memory, not a hard drive, but the startup process is similar.
  • Examples of firmware include the Bootstrap program, Bootloader, BIOS, and operating systems in embedded devices.
  • Booting up your computer is like waking up a sleepy robot. First, its ‚Äúalarm clock‚Äù (bootstrap) nudges it awake. Then, the robot remembers basic routines (firmware), and finally, it puts on clothes and starts working (OS loads and applications run).

4.1d

  • An interrupt is a signal from hardware or software that temporarily stops a processor‚Äôs current task to handle the interrupt.
  • Interrupts can be categorized into software and hardware interrupts:
    • A common hardware interrupt example is pressing a key on the keyboard or clicking/moving a mouse.
    • Software interrupts examples include division by zero, simultaneous memory access by processes, or software requesting user input.
    • Hardware interrupts examples include pressing keyboard keys, mouse clicks, hardware errors, or connecting new devices.
  • Interrupts have different priority levels. High-priority interrupts (e.g., hardware failures) are addressed immediately, whereas low-priority interrupts (e.g., key presses) are queued.
  • When an interrupt occurs:
    • It is placed in the interrupt priority queue. The processor finishes its current fetch-decode-execute cycle and then checks the queue.
    • If a higher priority interrupt is detected, the current process is saved on the stack, the interrupt source is checked, and the appropriate Interrupt Service Routine (ISR) is called.
    • Once the ISR completes, the processor retrieves the previous process from the stack and continues execution.
    • The processor saves the current task status (program counter and registers).
    • It executes the interrupt service routine (ISR) to address the interrupt.
    • After completion, it restores the saved status to continue the interrupted task.
  • Buffers are temporary memory storage used alongside interrupts to manage data transfer between fast and slow devices, allowing smoother multitasking.
  • Buffer use: When downloading media while playing it simultaneously, buffers temporarily store incoming data. This ensures continuous playback even if data arrives unevenly, providing a smooth user experience.
  • Printing process: Printing uses buffers to temporarily store print data and interrupts to manage data flow. This allows the processor to perform other tasks while the printer processes data, avoiding idle processor time and enhancing efficiency.
  • Interrupts are like someone shouting ‚ÄúPizza delivery!‚Äù during a boring lecture. The teacher (CPU) pauses the topic, checks the door, handles the delivery (ISR), and then goes back to teaching as if nothing happened.
