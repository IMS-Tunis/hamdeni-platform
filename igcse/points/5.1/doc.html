5.1 The Internet and the WWW

Computer Science IGCSE notes, prepared by Dr. Hamdeni

5.1a — Topic

  • The Internet is the infrastructure that links millions of devices together, allowing them to communicate and exchange data.
  • The internet is a type of Wide Area Network (WAN).
  • The World Wide Web is just one service provided by the internet. It lets users access websites using a browser.
  • The internet allows other services too, such as:
    • Sending and receiving Email.
    • Real-time communication like text, voice, and video chats.
  • The World Wide Web allows access to websites, each made up of Web Pages grouped into Websites.
  • A helpful analogy is:
    • The Internet is like a system of roads and highways.
    • The World Wide Web are like delivery trucks using those roads to transport goods (data).
  • This example compares sending an email versus browsing a website. Sending an email uses the Internet, while browsing www.bbc.co.uk/news uses the World Wide Web. This illustrates that the web is just one service on the internet.
  • Think of the Internet as a pizza delivery road system, and the World Wide Web as the pizzas being delivered. No roads = no pizzas!

5.1b — Topic Summary

  • A Uniform Resource Locator (URL) is what you type into the browser to open a website.
  • The URL guides the browser to the correct server and file. It usually contains the following parts:
    • Protocol.
    • Domain Host.
    • Domain Name.
    • Domain Type}.
    • File Name.
  • This example breaks down the parts of a URL using https://www.hoddereducation.co.uk/ict. "https://" is the Protocol. "www" is the Domain Host. "hoddereducation" is the Domain Name. ".co.uk" is the Domain Type. "/ict" is the Path.
  • A URL is like giving your browser a GPS address. Without it, your browser just sits there waiting like a lost tourist asking, "Where do I go?"

5.1c — Topic Summary

  • The Hypertext Transfer Protocol (HTTP) allows the browser to request and load websites.
  • HTTPS (Hypertext Transfer Protocol Secure) protects sensitive information, especially on login or payment pages.
  • Standard HTTP does not protect personal information like passwords during transmission. HTTPS is used instead to prevent hacking and data theft.
  • HTTPS uses encryption protocols to protect data.
    • SSL (Secure Socket Layer).
    • TLS (Transport Layer Security).
  • The SSL works using public key and private key encryption, which means that only the correct receiver can unlock and read the information.
  • The operation of HTTP is simple: the browser sends a request for a page using HTTP. The server sends back HTML and resources, and the browser displays them.
  • HTTPS follows these steps to create a secure connection between browser and server:
    • The browser asks the server for its Digital Certificate.
    • The server sends a certificate issued by a Certificate Authority.
    • The browser checks if the certificate is valid and trusted.
    • If valid, a secure encrypted connection is created.
    • If not, the browser shows a warning.
  • Users can check if a website is secure by looking for “https://” at the beginning of the URL and checking for a padlock symbol 🔒 near the address bar.
  • This example shows how to recognize a secure site. You can identify a secure website by the "https://" in the address and a padlock icon 🔒 in the browser's address bar. This confirms that the data is encrypted.
  • HTTP is like sending a postcard — everyone can read it! HTTPS is like sending a sealed envelope with a bodyguard.

5.1d — Topic Summary

  • A Web Browser is what we use to load and read online content.
  • Browsers request website files, including:
    • HTML (Hypertext Markup Language).
    • CSS (Cascading Style Sheets).
    • JavaScript.
  • The browser Renders this code so users can interact with the website visually.
  • It is important to know the difference between:
    • A Web Browser.
    • A Search Engine.
  • A Web Browser includes several key features that enhance usability:
    • Address Bar.
    • Tab.
    • Bookmark.
    • User History.
    • Navigation Tools.
    • Cache.
    • Hyperlinks.
    • Cookies.
  • A Web Browser is like a magical window into the internet, but don’t confuse it with a search engine. That’s like thinking your kitchen window is the same as your GPS!

5.1e — Topic Summary

  • When a user enters a URL into the Address Bar, the browser begins a series of steps to load the page.
  • The browser uses the Domain Name System (DNS) to find the website's location.
    • If the DNS has the answer, it returns the matching IP Address.
    • If the first DNS doesn’t have the answer, it passes the request to another DNS until one finds it.
  • The Web Server responds to the browser’s request.
    • It sends back the website’s HTML, CSS, and JavaScript files.
    • The browser Renders the content visually for the user.
  • This example shows a DNS lookup process. You type www.hoddereducation.co.uk. DNS returns 107.162.140.19. The browser uses this IP address to contact the web server and request the page content.

5.1f — Topic Summary

  • A Cookie holds information as key–value pairs, such as (language, English) or (theme, dark).
  • Cookies are created when the Web Server sends an empty cookie file to the browser. Then, the user’s details are added to the cookie file, which is saved on the device.
  • Cookies are used for various tasks that personalise the user experience:
    • Save login information like usernames or email addresses.
    • Keep shopping cart items when buying online.
    • Track browsing history for personalised ads.
    • Store settings such as language or display preferences.
    • Save game progress or quiz scores.
    • Recognise returning users and load their preferences.
  • Session Cookie is useful for tasks that only last for one visit.
  • Persistent Cookie helps websites remember you across multiple visits.
  • Cookies can pose privacy risks if misused.
    • They may be saved without the user’s knowledge.
    • Unauthorized websites might access them.
    • Hackers could intercept cookies during transmission.
    • Malicious actors can use them to build false profiles or steal identities.
  • While useful, cookies raise privacy concerns for some users because they build up a detailed profile of online habits. You can manage or block cookies in your browser settings.
  • This example illustrates session cookies. When you add items to an online shopping cart, a session cookie stores the items. If you close the browser, the cart will be empty when you return because the session cookie was deleted.
  • This example illustrates persistent cookies. If you tick "Remember Me" when logging into a site, a persistent cookie is created. It stores your login so the site can recognize you next time.
  • Cookies aren’t edible, but they do help websites "remember" who you are, like your favourite robot waiter that always knows your order.
