2.1 Types and methods of data transmission

Computer Science IGCSE notes, prepared by Dr. Hamdeni

2.1a: Data is broken down into packets

  • Data sent over long distances is divided into data packet, sometimes called datagrams, where datagrams is another word for packets; smaller units are easier to control than one continuous stream.
    • Using packets allows different routes to be used if one route is busy or unavailable, and the packets are reassembled at the destination into the original message.
  • Photograph split into packets: when a large photo is sent from one computer to another, it is divided into many packets that travel separately and are reassembled at the destination to rebuild the original image.
  • Packets are like puzzle pieces from the same picture. Each piece might travel by a different road, but the photo only appears when all pieces arrive and are put together in order.

2.1b: Structure of a packet

  • A typical data packet has three parts: a packet header, the payload, and a trailer.
    • The packet header includes source and destination IP addresses, protocol identification, and sequencing information. It also carries a hop limit that is decreased by each router; if it reaches zero before delivery, the packet is discarded.
    • The payload is the actual data carried by the packet; its size is not fixed and depends on the network technology and protocol in use.
    • The trailer marks the end of the packet and includes an error-checking method such as a cyclic redundancy check (CRC) in which the sender stores a value derived from the payload bits, the receiver recomputes this value, and any mismatch causes a re-send.
  • A packet is like a posted letter. The header is the address label, the payload is the letter inside, and the trailer is the seal that lets the receiver check nothing went wrong on the journey.

2.1c: Process of packet switching

  • packet switching splits a message into packets that are sent independently across the network and then reassembled in the correct order at the destination using header information.
    • At each stage, routers, which are devices that read the packet header and forward the packet to the next node, choose the next route based on current conditions such as queue lengths, selecting a shortest available path that may differ from the absolute shortest path if parts are busy or unsuitable.
    • Packets can arrive out of order, and the destination system reorders them correctly.
  • If a packet is missing or corrupted, the destination requests that packet again so the full message can be rebuilt correctly.
  • Benefits include avoiding the need to tie up a single line, the ability to reroute around failures or congestion, usage can expand easily, and high data rates; drawbacks include possible packet loss requiring re-sending, delays from reordering, and challenges for real-time streaming.
  • Different paths in practice: one packet may go through Router A while another goes through Router B, so they arrive in a different order and are then put back into the correct order at the destination.
  • Video call delays: during a live video call, sound and video are sent as packets. If parts of the network are busy, some packets take alternate routes or need re-sending, so they arrive late and cause short pauses or out-of-sync audio and video.
  • Routers behave like map apps that send cars along less busy streets. Different cars from the same group can take different roads, and the parking attendant at the end puts them back in the right order.

2.1d: Describe how data is transmitted from one device to another using different methods of data transmission

  • Serial transmission sends data one bit at a time over a single wire or channel.
    • Serial works well over long distances and keeps received data synchronised because there is only one channel, although the data rate is lower than parallel transmission.
  • Parallel transmission sends several bits at the same time over multiple wires or channels, often sending a whole byte, where a byte is a group of bits treated as a single unit.
  • simplex transmission sends data in one direction only from sender to receiver.
  • half-duplex transmission allows data to be sent in both directions, but not at the same time.
  • full-duplex transmission allows data to be sent in both directions at the same time.
  • Universal Serial Bus (USB) between a computer and a printer sends bits serially, which simplifies cabling.
  • Directions are like roads: simplex is a one-way street, half-duplex is a single-lane bridge with traffic lights, and full-duplex is a two-way road. Bit flow is like getting through a doorway: serial is one person at a time through one door, parallel is several doors side by side that work well only when the corridor is short.
  • Computer to printer: the computer sends print data to the printer only, showing one-way flow and simple control.
  • Walkie-talkie: a user presses to talk and releases to listen, showing two-way communication with turn-taking.
  • Home broadband connection: upload and download occur simultaneously, which reduces waiting and improves interactive use.

2.1e: Suitability of each method (advantages and disadvantages)

  • Serial transmission is used for long distances, is less expensive than parallel because it needs fewer hardware lines, and maintains synchronisation at a lower data rate than parallel.
  • Parallel transmission is preferred for short distances and time-sensitive transfers because it can be faster, but it needs more hardware and is therefore more expensive.
  • USB connection between a computer and a peripheral: serial transmission over a single cable, suitable for longer cable runs than parallel links used only over short distances.
  • Quick rule of thumb: for long journeys choose serial; for short internal links choose parallel. Choose simplex, half-duplex, or full-duplex based on how much two-way talking the task needs.

2.1f: Universal Serial Bus (USB) interface

  • Universal Serial Bus (USB) uses serial data transfer, and some USB connection operate in half-duplex or full-duplex modes, meaning they send in both directions either not at the same time or at the same time.
  • Selected benefits of Universal Serial Bus (USB) are automatic device detection and setup, keyed connectors that fit only one way, support for different data rates from about 1.5 megabits per second (Mbps) to about 5 gigabits per second (Gbps), and provision of a +5 volt power supply through the cable.
  • Selected drawbacks of Universal Serial Bus (USB) are a standard cable length of about 5 metres unless hubs are used, possible lack of support for very early versions on new computers, and data transfer that can be slower than Ethernet on comparable systems.
  • For error handling, the Universal Serial Bus (USB) protocol requests re-sending when errors are detected so that the final received data is error-free.
  • USB-C is a form of Universal Serial Bus (USB) with a 24-pin symmetrical connector that can be inserted either way and supports up to about 100 watts at 20 volts for power delivery, data rates up to about 10 Gbps, and high-resolution video such as 4K, while remaining backward compatible when a suitable adaptor is used.
  • USB memory stick transfer: copying files between a USB flash drive and a computer uses Universal Serial Bus (USB) serial transfer, which shows how devices are detected automatically and data moves through a simple one-way-fit connector.
  • USB-C is like a reversible key that fits either way. A USB cable can also carry a little power, so one wire can both move data and keep small devices powered.
