3.3a: Primary storage: RAM and ROM

  • primary storage is the part of a computerâ€™s memory that can be accessed directly by the CPU through a direct connection.
    • It includes two main types: random access memory (RAM) and read only memory (ROM).
  • RAM is a volatile type of storage, which means that all data is lost when the power is turned off.
    • It can be read from and written to, and both the user and the computer can change its contents.
    • It stores data, files, and parts of programs or the operating system that are currently in use.
    • The larger the size of RAM, the faster the computer can work because it needs to access slower secondary storage less often.
    • RAM can be upgraded by adding more RAM components when the computer supports expansion.
    • RAM allows any memory location to be accessed directly and quickly, which is why it is faster than secondary storage.
  • Two main types of RAM exist: dynamic RAM (DRAM) and static RAM (SRAM).
    • DRAM stores each bit of data in a tiny capacitor, with a transistor controlling reading and writing. It must be refreshed regularly to keep its data.
    • SRAM uses flip flops to store each bit of data, does not need refreshing, and is faster than DRAM.
    • DRAM is cheaper and has higher capacity, so it is used for main memory. SRAM is used for very fast memory areas such as the CPU cache.
  • ROM is a non volatile type of storage, meaning it keeps its contents when power is turned off.
    • Its contents are fixed and cannot normally be changed by the user or by applications.
    • It is used to store start up instructions, such as the bootstrap and the BIOS, which the computer needs when it is first switched on.
  • Key differences between RAM and ROM are:
    • RAM is temporary, volatile, and can be written to and read from.
    • ROM is permanent, non volatile, and can only be read.
    • RAM stores working data, while ROM stores fixed data needed for starting the system.
  • Remote controlled toy car: In such a device, ROM can store factory settings and basic control routines, while RAM can temporarily store instructions received from the remote control. This shows why devices need both fixed start up data and temporary working memory.
  • Think of RAM like your desk space â€” the bigger it is, the more papers you can spread out without having to constantly go to the filing cabinet.

3.3b: Secondary storage

  • Secondary storage refers to storage devices that are not directly accessed by the CPU.
    • Data and instructions from secondary storage must first be transferred into primary storage before processing.
    • Secondary storage can be internal or external to the computer.
  • Secondary storage is non volatile, keeping data even when the power is switched off.
  • It can store more data than primary storage, but it is slower to access.
    • All applications, the operating system, and user files such as documents, photos, and music are stored on secondary storage.
  • Saving to a USB memory stick: Files saved to a USB stick remain available after unplugging it, while unsaved work in RAM is lost when power is off. This illustrates non volatility of secondary storage.
  • Secondary storage is like a giant basement: you can keep everything there for years, but it takes longer to climb down and find it compared to grabbing something from your desk (RAM).

3.3c: Magnetic, optical, and solid state storage

  • magnetic storage uses magnetism to store data on spinning circular disks called platters.
    • Platters spin at high speed, and read or write heads with electromagnets move across the surfaces to store or retrieve data.
    • The surfaces are organised into tracks and sectors, with each sector holding a fixed amount of data.
    • A magnetised spot represents binary 1, and a demagnetised spot represents binary 0.
  • Hard disk drives are a common type of magnetic storage.
    • They can suffer from latency, the delay while the disk rotates the required data under the read or write head.
    • Files can become fragmented, with parts stored in non adjacent sectors, slowing down access. Defragmentation software can rearrange the data to speed things up.
  • optical storage uses laser light to read and write data on circular discs.
    • Data is stored as pits and lands along a single spiral track from the centre of the disc to the edge.
    • CDs and DVDs use a red laser, while Blu ray discs use a blue laser with a shorter wavelength, allowing smaller pits and higher capacity.
    • DVDs can be dual layer, with two recording layers separated by a spacer to increase storage capacity. Blu ray discs can be single or dual layer, with capacities up to 50 GB.
    • Laser wavelengths and format examples: CD uses red light at about 780 nm, DVD uses red light at about 650 nm, and Blu ray uses blue light at about 405 nm. The shorter wavelength allows more data to fit.
  • Solid state storage has no moving parts and uses semiconductor chips to store data.
    • Data is stored in cell made of transistors arranged in a grid, with each cell containing a control gate and a floating gate separated by a dielectric layer.
    • The floating gate holds an electrical charge for binary 1 or no charge for binary 0.
    • Solid state storage can use NAND or NOR structures to connect the cells.
    • Solid state storage is non volatile; it keeps data even when power is off.
  • Examples of solid state storage devices include solid state drives and USB flash drives.
  • Advantages of solid state drives over hard disk drives include:
    • Faster access speeds and no rotational latency.
    • Greater reliability because there are no moving parts.
    • Lower power use and cooler operation, suitable for laptops.
    • Lighter weight and smaller size.
  • A disadvantage of solid state drives is limited write endurance.
    • Data must be erased before rewriting to the same location.
  • SSD vs HDD in one sentence: Because SSDs use floating gate transistors and no moving parts, they are typically faster and lighter than hard disk drives.
  • SSDs are like superhero messengers â€” they run everywhere instantly without waiting for elevators or spinning disks.

3.3d: Virtual memory

  • virtual memory is created when part of a computerâ€™s hard disk drive or solid state drive is used as an extension of RAM.
    • It is used when RAM becomes full, allowing more programs to run at the same time.
  • Data that is not immediately needed in RAM is moved to virtual memory in units called page.
    • When the data is needed again, it is moved back into RAM.
  • Benefits of virtual memory include:
    • Ability to run programs larger than the available physical RAM.
    • More efficient use of RAM by moving unused data out temporarily.
    • Reduced need to install extra RAM.
  • Drawbacks of using hard disk based virtual memory include slower performance because of disk thrashing.
    • Disk thrashing occurs when frequent swapping between RAM and virtual memory causes a high number of read or write movements, reducing processing speed.
    • Excessive swapping, called disk thrashing, can also shorten the life of mechanical drives.
    • Using a solid state drive instead of a hard disk for virtual memory can reduce thrashing.
    • Adding more RAM or using a solid state drive can reduce virtual memory problems.
  • Five program scenario: If five programs fill RAM, the least recently needed data can be paged to virtual memory so a new task can start without a crash. When that data is needed again, its page returns to RAM.
  • Virtual memory is like borrowing your neighborâ€™s garage when yours is full â€” convenient, but walking back and forth takes extra time.

3.3e: Cloud storage

  • cloud storage stores data on remote server owned and maintained by a hosting company.
    • Data is often stored on multiple servers to ensure it can still be accessed during maintenance or in case of hardware failure.
    • Cloud servers are often in large, secure data centers and operate 24/7.
  • Three main types of cloud storage are:
    • Public cloud, where the provider and the customer are different organisations.
    • Private cloud, a dedicated storage environment within a company infrastructure.
    • Hybrid cloud, a combination of public and private cloud systems.
  • Advantages of cloud storage include:
    • Data can be accessed from anywhere with an internet connection.
    • There is no need to carry physical storage devices.
    • Cloud storage offers remote backups that help with disaster recovery.
    • Storage space can be scaled up or down easily.
    • Cloud providers handle security and backups.
  • Disadvantages of cloud storage include:
    • Dependence on internet speed and stability.
    • Possible ongoing costs for large storage or high data transfer.
    • Risks from security breaches, hacking, or provider failure.
  • Google Drive accessibility: Photos saved to a cloud service such as Google Drive can be viewed on any device that signs in, while photos stored only on a local phone are not available on other devices without copying.
  • Cloud storage is like having a magical backpack â€” you can open it from anywhere in the world, but only if thereâ€™s Wi-Fi in the air.

3.3f: Advantages and disadvantages of cloud vs local storage

  • Advantages of cloud storage over local storage include:
    • Access from any location with internet access.
    • No need to buy and maintain physical storage hardware.
    • Easier recovery after a disaster.
  • Disadvantages of cloud storage compared to local storage include:
    • No access to data without internet.
    • Possible security and privacy concerns.
    • Potential ongoing costs.
  • Advantages of local storage compared to cloud storage include:
    • Full control over security and backups.
    • No ongoing costs for storage services in local storage.
    • No internet is needed to access the data.
  • Disadvantages of local storage compared to cloud storage include:
    • Cost of buying and maintaining hardware.
    • Storage space is fixed and may be unused if needs decrease.
