1. What is an array?

  • An array is a data structure that stores several data items under one identifier.
  • All elements in one array must have the same data type.
  • Each element is accessed using an index, which identifies its position in the array.
  • Arrays are useful when a program needs to store multiple related values in a uniformly accessible way.
Example: Accessing one element in an array
Colour[0] ← "red"

This stores the value "red" in the first position of the array Colour.

2. One-dimensional arrays

  • A one-dimensional array can be thought of as a list.
  • It has one index only.
  • When a one-dimensional array is declared in pseudocode, the declaration includes:
    • the array name
    • the first index value
    • the last index value
    • the data type
Example: Declaring a one-dimensional array
DECLARE MyList : ARRAY[0:9] OF INTEGER

This declares an integer array called MyList with ten elements, indexed from 0 to 9.

Example: Storing one value in a one-dimensional array
MyList[4] ← 27

This stores the value 27 at index 4.

Example: Reading values into a one-dimensional array
FOR Counter ← 0 TO 9
OUTPUT "Enter next value "
INPUT MyList[Counter]
NEXT Counter

The loop uses the variable Counter to fill every position in the array.

Example: Outputting one value from a one-dimensional array
OUTPUT MyList[1]

This outputs the value stored at index 1.

3. Two-dimensional arrays

  • A two-dimensional array can be thought of as a table.
  • It uses rows and columns.
  • Each element needs two indexes: one for the row and one for the column.
  • The first element of a two-dimensional array may be at position 0,0 if zero indexing is used.
Example: Declaring a two-dimensional array
DECLARE MyTable : ARRAY[0:9,0:2] OF INTEGER

This declares an integer table with row indexes from 0 to 9 and column indexes from 0 to 2.

Example: Storing one value in a two-dimensional array
Data[4, 3] ← 10

This stores the value 10 in one position identified by two indexes.

Example: Outputting one value from a two-dimensional array
OUTPUT(ArrayData[4, 1])

To access one element in a two-dimensional array, both indexes must be given.

4. First index: zero or one

  • The first element of an array can have an index of zero or one.
  • This depends on the programming language being used.
  • Most programming languages automatically set the first index to zero.
  • When writing or reading an algorithm, always check clearly whether the first index is 0 or 1.
Example: Zero-indexed and one-indexed notation
Array[0]
StudentName[1]

Both styles are valid. The important point is to know which indexing system is being used.

5. Using variables as array indexes

  • A variable can be used as the index of an array.
  • This is very common inside a loop.
  • Using a loop counter as the index makes it possible to process all elements of an array one by one.
Example: Using a variable as an array index
FOR Counter ← 1 TO 10
OUTPUT "Enter Name of Student "
INPUT StudentName[Counter]
NEXT Counter

Here, Counter changes each time through the loop and selects a different position in the array.

6. Nested iteration with arrays

  • Two-dimensional arrays usually need nested loops.
  • One loop is used for one index and the second loop is used for the other index.
  • This allows the program to read or write every element in the table.
Example: Populating a two-dimensional array using nested loops
FOR ColumnCounter ← 0 TO 2
FOR RowCounter ← 0 TO 9
OUTPUT "Enter next value "
INPUT MyTable[RowCounter, ColumnCounter]
NEXT RowCounter
NEXT ColumnCounter

This uses two counters because a two-dimensional array has two indexes.

7. Meaningful identifiers for arrays

  • In a maintainable program, identifiers should be meaningful.
  • This includes array names.
  • A meaningful array name helps another programmer understand what the array stores.
  • For example, a name such as StudentName or StudentMark is clearer than a name such as A or X.
Example: Clear array names
DECLARE StudentName : ARRAY[1:10] OF STRING
DECLARE StudentMark : ARRAY[1:10] OF INTEGER

These names immediately show what each array is intended to store.
