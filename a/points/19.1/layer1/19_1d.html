19.1d:
Show how it is possible for ADTs to be implemented from another ADT:
  • Describe the following ADTs and demonstrate how they can be implemented from appropriate builtin types or other ADTs: stack, queue, linked list, dictionary, binary tree
  • An Abstract Data Type (ADT) is defined by two parts: the data it stores and the operations that act on that data. An ADT definition can use existing data types inside it.
  • The same ADT can be implemented in different ways (for example, using arrays, records, linked lists, or a combination). The choice affects how you store items and how you update the structure.

1) Stack: common implementation choices

A) Stack implemented using an array

  • The stack items are stored in a fixed-size 1D array.
  • A pointer (often called TopPointer) stores the index of the current top item.
  • Push increases TopPointer; Pop decreases it.
  • Because the array has a fixed size, the stack can become full (this causes overflow when pushing).

Example: if TopPointer = 3, then the top item is stored in Stack[3].

B) Stack implemented using a linked list

  • Stack items are stored as nodes linked together using pointers.
  • A pointer identifies the first node (this node represents the top of the stack).
  • Push inserts a node at the front of the list.
  • Pop removes the first node of the list.
  • This avoids a fixed maximum size (limited only by available memory).

Key idea: in a linked list stack, push/pop work by changing a pointer, not by shifting data.

2) Queue: common implementation choices

A) Queue implemented using an array

  • The queue items are stored in a fixed-size 1D array.
  • Two pointers are used:
    • HeadPointer – index of the front item (next to remove).
    • TailPointer – index of the rear item (most recently added).
  • A counter such as NumberItems stores how many items are currently in the queue.
  • Because the array size is fixed, the queue can become full (this causes overflow when enQueueing).
  • A circular (wrap-round) approach is often used so pointers can reuse freed spaces.

Example: if HeadPointer = 2 and TailPointer = 5, the next deQueue reads from Queue[2].

B) Queue implemented using a linked list

  • Queue items are stored as linked nodes.
  • Two pointers are used:
    • a pointer to the front node (for deQueue)
    • a pointer to the rear node (for enQueue)
  • enQueue adds a node at the rear.
  • deQueue removes a node at the front.
  • This avoids a fixed maximum size (limited only by memory).

3) Linked list: common implementation style used for algorithms

  • A linked list can be implemented using an array of records.
  • Each node contains two fields:
    • Data – the value stored in the node.
    • Pointer – the index of the next node.
  • A special value such as -1 is used as NullPointer to mean “no next node”.
  • StartPointer stores the index of the first node in the list.
  • Unused array positions are managed using a free list, whose first index is stored in FreeListPtr.
  • Free nodes are linked together so their space can be reused.

Example: if StartPointer = 4, then the first node of the list is stored at index 4.

In an ordered linked list, insertion first finds the correct position, then updates pointer values to link the new node either:

  • at the start of the list, or
  • between two existing nodes.

4) Dictionary: implemented using a hash table

A) Dictionary as key–value pairs

  • A dictionary stores key–value pairs.
  • The key is used to look up the value.
  • Example: key "book" maps to value "livre".
  • Many languages provide a built-in dictionary type.

Example in Python: EnglishFrench["book"] = "livre" adds a pair, and EnglishFrench["book"] retrieves the value.

B) Implementation using a hash table

  • A dictionary can be implemented using a hash table to allow fast direct access.
  • A hash table is a 1D array where the array index (address) is calculated from the key using a hash function.
  • Example hash function: Address ← Key MOD n
  • If two different keys produce the same address, this is a collision.
  • Common collision handling methods:
    • Chaining – store colliding items in a linked list at that address.
    • Open addressing – search nearby slots to find an empty position.

5) Binary tree: implemented using an array of records

  • A binary tree can be stored in an array of records.
  • Each node contains three fields:
    • LeftPointer – index of the left child
    • Data – value stored in the node
    • RightPointer – index of the right child
  • Pointers are stored as integer indexes.
  • -1 is used as the NullPointer to indicate “no child”.
  • RootPointer stores the index of the root node.

Example: if RootPointer = 0, the root node is stored at index 0.
