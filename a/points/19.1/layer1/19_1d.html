  • An Abstract Data Type (ADT) is defined by two parts: the data it stores and the operations that act on that data. An ADT definition can use existing data types inside it.
  • The same ADT can be implemented in different ways (for example, using arrays, records, linked lists, or a combination). The choice affects how you store items and how you update the structure.

Implementing ADTs from Built-in Types and Other ADTs

An Abstract Data Type (ADT) defines the behaviour of a data structure (its operations and rules), but it does not define how the data is stored internally.

The same ADT can therefore be implemented in different ways:

  • Using a built-in data type (such as an array).
  • Using another ADT as the underlying structure.

When we say “implement an ADT using another ADT”, we mean: the operations of the first ADT are carried out using the operations of the second ADT.

1) Stack ADT

A stack follows the LIFO (Last-In, First-Out) principle. Main operations: Push and Pop.

A) Stack implemented using a built-in array

  • Items are stored in a 1D array.
  • TopPointer stores the index of the current top item.
  • Push → increase TopPointer, then store the new value.
  • Pop → read the top value, then decrease TopPointer.
  • The array has a fixed size, so overflow can occur.

B) Stack implemented using the Linked List ADT

  • Items are stored as nodes in a linked list.
  • StartPointer represents the top of the stack.
  • Push → use the linked list insert-at-start operation.
  • Pop → use the linked list delete-from-start operation.
  • No fixed maximum size (limited only by memory).

Here, the Stack ADT is implemented using the Linked List ADT. Stack operations are performed using linked list operations.

2) Queue ADT

A queue follows the FIFO (First-In, First-Out) principle. Main operations: enQueue and deQueue.

A) Queue implemented using a built-in array

  • Items are stored in a 1D array.
  • HeadPointer identifies the front item.
  • TailPointer identifies the rear item.
  • A counter such as NumberItems tracks how many elements are stored.
  • A circular (wrap-round) method allows reuse of freed space.
  • The array has a fixed size, so overflow can occur.

B) Queue implemented using the Linked List ADT

  • Items are stored as nodes in a linked list.
  • A pointer to the front and a pointer to the rear are used.
  • enQueue → use linked list insert-at-end.
  • deQueue → use linked list delete-from-start.
  • No fixed maximum size (limited only by memory).

Here, the Queue ADT is implemented using the Linked List ADT.

3) Linked List ADT

A linked list stores items in nodes connected by pointers.

Implementation using a built-in array (array-based linked list)

  • The list is stored in an array of records (2D list).
  • Each node contains:
    • Data
    • Pointer (index of next node)
  • NullPointer = -1 means “no next node”.
  • StartPointer stores the first node index.
  • FreeListPtr manages unused nodes using a free list.

Here, the Linked List ADT is implemented using a built-in array.

4) Dictionary ADT

A dictionary stores key–value pairs. The key is used to retrieve its associated value.

A) Using a built-in dictionary type

  • Many programming languages provide a built-in dictionary type.
  • Example in Python: EnglishFrench["book"] = "livre".

B) Using a Hash Table ADT

  • A dictionary can be implemented using a hash table.
  • The hash table itself is implemented using a 1D array.
  • A hash function converts a key into an array index.
  • If two keys map to the same index, a collision occurs.
  • Collisions can be handled using chaining, which uses the Linked List ADT.

Therefore:
Dictionary ADT → implemented using Hash Table ADT → collision handling may use Linked List ADT.

5) Binary Tree ADT

A binary tree is a hierarchical structure where each node has at most two children.

Implementation using an array of records

  • The tree is stored in an array.
  • Each node contains:
    • LeftPointer
    • Data
    • RightPointer
  • NullPointer = -1 represents “no child”.
  • RootPointer stores the index of the root node.

Here, the Binary Tree ADT is implemented using a built-in array.

Comparison Summary

The table below compares implementations using built-in types versus other ADTs.

ADT Built-in Type Using Another ADT Main Difference
Stack Array + TopPointer Linked list (insert/delete at start) Array is fixed size; linked list can grow
Queue Array + wrap-round Linked list (rear insert, front delete) Array needs circular logic
Dictionary Built-in dictionary Hash table (+ linked list for chaining) Uses hashing for fast access
