1) Insertion sort (algorithm idea)

  • Insertion sort treats the list as having a sorted part and an unsorted part. Each new item from the unsorted part is inserted into its correct position in the sorted part.
  • The loop takes the next item (ItemToBeInserted), then shifts larger sorted items one position to the right until the correct insertion position is reached, then inserts the item.
  • See Annex: InsertionSortPseudo and Annex: InsertionSortPython.

2) Bubble sort (algorithm idea)

  • Bubble sort repeatedly compares adjacent items and swaps them if they are in the wrong order.
  • An inner loop performs adjacent comparisons and swaps; a variable (Unsorted) is reduced each outer pass, so the algorithm does fewer comparisons as the largest items move into their final positions.
  • See Annex: BubbleSortPseudo and Annex: BubbleSortPython.

3) Performance depends on initial order and number of items

How fast a sorting method works depends on two things: how many items there are and how the data is arranged at the start.

A. Effect of the number of items

  • When the list has more items, the sorting method must do more repeated steps, so the sorting time increases.
  • Bubble sort repeats many comparisons (and swaps when needed). The key repeated check is: IF MyList[j] > MyList[j + 1].
  • Example (bubble sort, worst case): the number of comparisons grows quickly as n increases.

B. Effect of the initial order of the data

  • To test performance properly, change the starting order of items (already in order, reverse order, mixed) and observe the effect.
  • both algorithms are fast on sorted data and slow on reversed data, but on partially sorted data insertion sort typically performs better because it can place an item into its correct position in one insertion step, while bubble sort moves items gradually via neighbour swaps across multiple passes.:
Example

List: [1,2,3,4,6,7,8,9,5]

Insertion sort

  • When it reaches the last item 5, it inserts it after 4 by shifting 6,7,8,9 right once.
  • Sorted immediately: [1,2,3,4,5,6,7,8,9] (done in one outer step)

Bubble sort

  • 5 moves left only one position per pass (neighbour swaps):
    • Pass 1: [1,2,3,4,6,7,8,5,9]
    • Pass 2: [1,2,3,4,6,7,5,8,9]
    • Pass 3: [1,2,3,4,6,5,7,8,9]
    • Pass 4: [1,2,3,4,5,6,7,8,9]

Point: insertion places 5 correctly in one step; bubble needs several passes because it keeps finding a new swap each pass.
