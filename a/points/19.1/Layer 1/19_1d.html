19.1d:
Show how it is possible for ADTs to be implemented from another ADT:
  • Describe the following ADTs and demonstrate how they can be implemented from appropriate builtin types or other ADTs: stack, queue, linked list, dictionary, binary tree
  • An Abstract Data Type (ADT) is defined by two parts: the data it stores and the operations that act on that data. An ADT definition can use existing data types inside it.
  • The same ADT can be implemented in different ways (for example, using arrays, records, linked lists, or a combination). The choice affects how you store items and how you update the structure.

1) Stack: common implementation choices

A) Stack implemented using an array (built in type)

  • The stack items are stored in a 1D array.
  • A pointer/index is used to indicate where the current “top” of the stack is.
  • Because an array has a fixed size, this implementation can become full.

B) Stack implemented using a linked list (ADT implemented from another ADT)

  • Stack items are stored as nodes, with each node pointing to the next.
  • A pointer identifies the first node (the current top of the stack).
  • Insert and delete only at the front of the list.
  • A linked list implementation can avoid a fixed maximum, which is one reason it can be better than an array implementation in some situations.

2) Queue: common implementation choices

A) Queue implemented using an array (built in type)

  • The queue items are stored in a 1D array.
  • Two pointers/indexes are typically used:
    • one for the front (where items are removed)
    • one for the rear (where items are added)
  • Because an array has a fixed size, this implementation can become full.

B) Queue implemented using a linked list (ADT implemented from another ADT)

  • Queue items are stored as nodes, with pointers linking nodes.
  • Pointers are used to identify the front and the rear of the queue (so items can be added at the rear and removed from the front).
  • Like stacks, linked list based queues can avoid a fixed maximum size.

3) Linked list: common implementation style used for algorithms

 A linked list can be implemented using an array of records, where each array element (node) has two fields: Data and Pointer.

 The Pointer field stores the index of the next node in the array; a special value such as -1 is used as a null pointer to mean “no next node”.

 The list is accessed using StartPointer, which stores the index of the first node in the list.

 Unused array positions are managed using a free list; its first node index is stored in FreeListPtr, and the unused nodes are linked together so their space can be reused.

 In an ordered linked list, insertion first finds the correct position, then links the new node either at the start or between two nodes by updating pointer values.

4) Dictionary: implemented using a hash table

A) Dictionary as key value pairs

  • A dictionary stores key value pairs; the key is used to look up the value. Example: key "book" maps to value "livre".
  • Many languages provide a built in dictionary type, for example Python:
    • EnglishFrench["book"] = "livre" adds a key value pair
    • EnglishFrench["book"] accesses the value for key "book"

B) How a dictionary is implemented: hash table (direct access)

  • A computer science dictionary is implemented using a hash table so that a value can be looked up using a direct access method.
  • A hash table is a 1D array of records, where an address (array index) is calculated from the record key (hashing).
  • Example hashing function: Address ← Key MOD(n + 1)
  • If two different keys hash to the same address, this is a collision.
  • Collision handling methods include:
    • Chaining: create a linked list for collisions starting at the hashed address
    • Closed hashing: store collisions in a separate overflow area
    • Open hashing: search neighbouring slots to find an empty slot

5) Binary tree: commonly implemented using an array of records

  • A binary tree can be implemented using an array, where each element is a record containing:
    • the data value
    • a left pointer
    • a right pointer
  • Pointers are often stored as integer indexes (pointing to positions in the array), and a special value such as -1 is used to indicate “no child” (null pointer).
