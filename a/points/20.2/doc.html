File-Processing Operations

File processing involves reading data from files and writing data to files. A file must be opened before use and closed after the operation is completed.

Opening a File

A file can be opened in different modes:

Read Mode ('r')
Open an existing file to read its content
Write Mode ('w')
Create a new file or overwrite an existing file
Append Mode ('a')
Open a file and add new data at the end
file = open("data.txt", "r")     # Open for reading
file = open("data.txt", "w")     # Open for writing
file = open("data.txt", "a")     # Open for appending
Choosing the Right Mode Choose the mode that matches your intention. Reading inspects existing content. Writing replaces content. Appending grows the file.

Reading a Record

Ways to read text from files:

readline()
readlines()
read()
Iteration
# Read one line at a time
file = open("data.txt", "r")
record = file.readline()
while record:
    # Process the record
    print(record.strip())
    record = file.readline()
file.close()
# Read all lines into a list
file = open("data.txt", "r")
records = file.readlines()
file.close()

# Process each record
for record in records:
    print(record.strip())
# Read the entire file as one string
file = open("data.txt", "r")
content = file.read()
file.close()

# Split content into lines
records = content.split("\n")
for record in records:
    if record:  # Skip empty lines
        print(record)
# Iterate directly over the file object
with open("data.txt", "r") as file:
    for line in file:
        # Process line as a record
        print(line.strip())
File Iteration Iteration over the file object processes lines in order and stops naturally at end-of-file. Note that readlines() often includes newline characters. strip() removes trailing newline characters when needed.

Complete Python Example: Student Marks Processing

# Open the file with student marks
markFile = open("stMarks.txt", "r")

# Initialize a list to store the student marks table
stmarkTable = []

# Loop through each line in the mark file
for row in markFile:

    # Split the line by comma to get individual records
    stRecord = row.split(",")

    # Extract student name and marks for each subject
    stName    = stRecord[0]
    mathMark  = int(stRecord[1])
    sciMark   = int(stRecord[2])
    geoMark   = int(stRecord[3])
    compMark  = int(stRecord[4])

    # Calculate the total and average of the marks
    total   = mathMark + sciMark + geoMark + compMark
    average = round(total / 4, 2)

    # Create a new record with the student's name,
    # marks, total, and average
    newstRecord = [stName, mathMark, sciMark,
                   geoMark, compMark, total, average]

    # Add the new record to the marks table
    stmarkTable.append(newstRecord)

    # Reset the record list for the next student
    newstRecord = []

# Close the mark file
markFile.close()

# Print the two-dimensional array of student marks
for row in stmarkTable:
    for element in row:
        print(element, end=" ")
    print()
Record Processing Each input line is treated as a record. Field conversion to integers enables arithmetic for totals and averages.

Writing a Record

# Writing a single record
file = open("data.txt", "w")
file.write("New record\n")
file.close()

# Writing multiple records
records = ["Record 1\n", "Record 2\n", "Record 3\n"]
with open("data.txt", "w") as file:
    file.writelines(records)
Newline Characters Include \n when a new record must start on a new line in the file. Without it, all records would be written on the same line.

Closing a File

# Manual closing
file = open("data.txt", "r")
content = file.read()
file.close()

# Using a context manager (recommended)
with open("data.txt", "r") as file:
    data = file.read()
# File is automatically closed here
Context Managers Automatic closing ensures resources are released even if an error occurs. This is the preferred way to work with files in Python.

File-Processing for Different File Organizations

Serial Files
Sequential Files
Random Files

Serial File

Records are stored in the order they are entered. Appending adds new records at the end. Access is naturally in arrival order.

# Appending to a serial file
with open("transactions.txt", "a") as file:
    file.write("Deposit: $100\n")
Serial Files Best when you mainly add new records and read everything in the order it was recorded.

Sequential File

Records are kept in a sorted order by a key, for example an ID. Reading proceeds from the beginning to the point of interest.

# Searching in a sequential file
with open("customers.txt", "r") as file:
    for line in file:
        name, customer_id = line.strip().split(",")
        if customer_id == "C105":
            print(f"Found customer: {name}")
            break
Sequential Files To keep the order after edits, the file is typically rewritten in the correct sequence.

Random File

Records are accessed in any order by computing a position from a key. The program moves the file pointer to that position, then reads or writes at that address. If two keys compute to the same position, a collision handling method is needed.

Random Access Files The key idea is position calculation, pointer movement, then direct access. Implementation details depend on the chosen addressing design.

Exception and Exception Handling

What is an Exception

An exception is an unexpected error during program execution, for example division by zero, opening a file that does not exist, or converting non-numeric input to an integer. If not handled, program execution stops at the error.

Why Use Exception Handling

Exception handling prevents crashes, allows controlled recovery, and displays clear messages for users. It improves robustness when operations might fail.

When to Use Exception Handling

Typical cases:

File Operations
Files may be missing or unreadable
User Input
User input may be invalid
Calculations
Calculations may fail (e.g., division by zero)
Resources
Resources may be unavailable

Python Exception Handling

Python uses try, except, and optionally finally.

try:
    # statements that might raise an exception
    pass
except ExceptionType:
    # handlers for specific exceptions
    pass
except ExceptionType2:
    # handlers for another type of exception
    pass
finally:
    # statements that always run
    pass

File Exception Example

try:
    with open("data.txt", "r") as file:
        content = file.read()
        print(content)
except FileNotFoundError:
    print("Error: The file was not found.")
except IOError:
    print("Error: Could not read the file.")
finally:
    print("Operation completed.")
Multiple Exception Handlers Multiple except blocks handle different error types. The finally block runs in all cases, whether an exception occurred or not. 
def division(num1, num2):
    try:
        result = num1 / num2
        print(f"Result: {result}")
    except ZeroDivisionError:
        print("Error: Division by zero is not allowed.")
    except TypeError:
        print("Error: Both arguments must be numbers.")

a = int(input("Enter num1: "))
b = int(input("Enter num2: "))
division(a, b)
Defensive Programming If an operation can fail, place it inside try and provide a clear response in except. This prevents crashes and improves user experience.
