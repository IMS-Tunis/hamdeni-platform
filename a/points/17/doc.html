17.a: Understanding How Encryption Works

Encryption, key concepts, security objectives, models, and quantum cryptography.

Encryption is the process of converting readable data (plaintext) into an unreadable format (ciphertext) to protect it during transmission. It ensures that sensitive information remains confidential, authentic, and unchanged during communication.

1.0 The Communication Story

  • A sender starts with plaintext (readable data).
  • The sender applies an encryption algorithm and a key to produce ciphertext (unreadable data).
  • The ciphertext is transmitted over a network, where an eavesdropper may try to intercept it.
  • The receiver applies the decryption algorithm and the correct key to recover the original plaintext.
  • Security goals include keeping data confidential, checking who sent it, and checking it was not changed.

1.1 Core Encryption Concepts

Plaintext

The original, readable data before encryption.

Ciphertext

The unreadable result after encryption.

Encryption Algorithm

The mathematical process used to convert plaintext into ciphertext.

Decryption Algorithm

The reverse process, converting ciphertext back into plaintext using a key.

Eavesdropper

An unauthorized person who intercepts data during transmission.

1.2 Security Objectives of Encryption

Confidentiality

Ensures that only the intended recipient can read the message.

Authenticity

Verifies the sender of the message and ensures they are legitimate.

Integrity

Guarantees that the data has not been altered during transmission.

Non-Repudiation

Prevents the sender or receiver from denying their involvement in the transmission.

1.3 Encryption Models

In practice, both symmetric and asymmetric encryption are used because they solve different problems: symmetric encryption is fast for large amounts of data, while asymmetric encryption helps avoid sharing a secret key in advance.

Key distinction

A fundamental distinction between symmetric encryption and asymmetric encryption is that symmetric encryption relies on one common key used by all communicating parties, whereas asymmetric encryption relies on a pair of keys in which one key is publicly available and the other must be kept secret.

1.3.1 Symmetric Key Encryption

  • Uses the same secret key for both encryption and decryption.
  • Suitable for fast and secure communication when both parties can share the key privately.

In symmetric encryption, the encryption algorithm can work in different ways depending on how it processes the data.

  • Block cipher is the encryption of a number of contiguous bits in one go rather than one bit at a time.
  • Stream cipher is the encryption of bits in sequence as they arrive at the encryption algorithm.
  • Block chaining is a form of encryption in which the previous block of ciphertext is combined with the block of plaintext and then encrypted, thus preventing identical plaintext blocks producing identical ciphertext.
Example

If the plaintext is "COMPUTER SCIENCE IS EXCITING" and a 10-digit key shifts each letter by certain positions, the ciphertext becomes "GQSMXYIV WKMGIRG MW IBAOXMQK."

Limitation

The key must be securely shared between sender and receiver. If intercepted, the encryption becomes useless. This is known as the key distribution problem.

Modern symmetric encryption, such as Advanced Encryption Standard (AES), uses 256-bit keys with 2^256 possible combinations, making it highly secure (this means an extremely large number of possible keys).

1.3.2 Asymmetric Key Encryption (Public Key Encryption)

Asymmetric encryption is often used to exchange a temporary secret key safely, then that temporary key is used for fast symmetric encryption of the main data.

Uses a pair of keys:

  • Public Key: Shared with everyone and used to encrypt data.
  • Private Key: Kept secret and used to decrypt data.
Example scenario
  • Mohamed wants to send a confidential message to Fatma.
  • To ensure only Fatma can read the message, Mohamed needs Fatma’s public key.
  • Fatma shares her public key with Mohamed. This key is safe to share with anyone.
  • Mohamed encrypts the message using Fatma’s public key.
  • He sends the encrypted message to Fatma.
  • When Fatma receives the message, she uses her private key to decrypt it and read the original content.
Why is this secure?

Even if someone intercepts the encrypted message, they cannot read it because they don’t have Fatma’s private key.
Only Fatma can decrypt the message since she is the only one who has the corresponding private key.

Advantages
  • Eliminates the need to share a secret key (key distribution problem).
  • Better security for large networks. Each person has a unique key pair, avoiding key distribution issues.

1.4 Advanced Encryption Approaches

1.4.1 Quantum Cryptography

Quantum cryptography uses the properties of photons (light particles) and quantum mechanics to enable ultra-secure communication.

Qubit

The basic unit of a quantum of information (quantum bit).

In some quantum communication systems, information can be represented using quantum states (for example, different photon polarizations), which is one way to think about how a qubit can be physically represented.

Polarization

Polarization is the direction in which a light wave oscillates.

Quantum Key Distribution (QKD)

In Quantum Key Distribution (QKD), photons are polarized to represent bits:

Vertical (|) = 1 Horizontal ( - ) = 0 45° Right (/) = 1 45° Left (\) = 0

A special method called Quantum Key Distribution (QKD) allows two people to share a secret encryption key while detecting any eavesdroppers. If a hacker tries to intercept the key, the act of measuring the photons changes their state, alerting the sender and receiver.

Advantages

Quantum cryptography can detect eavesdropping during key exchange because measuring certain quantum states changes them.

Limitations
  • Requires specialized hardware and dedicated quantum communication channels.

17.b: Secure Socket Layer (SSL) and Transport Layer Security (TLS)

Purpose, handshake process, and common applications of SSL/TLS.

SSL and TLS are protocols used to secure online communication by encrypting data and verifying the authenticity of websites.

Public Key Infrastructure (PKI)

A set of protocols, standards and services that allow clients and servers to authenticate each other using digital certificates issued by a CA.

2.1 Security Properties Provided by SSL/TLS

Encryption

Ensures that data transmitted between the client (browser) and the server (website) is unreadable by unauthorized parties.

Authentication

Confirms the identity of the website to prevent impersonation.

Data Integrity

Ensures that data has not been altered during transmission.

2.2 SSL/TLS Handshake Process

  1. The client connects to a secure website via HTTPS.
  2. The server sends an SSL certificate, which contains its public key, to the client.
What is the certificate in this step?

A digital certificate is an electronic document used to prove the identity of a website or individual. It contains a public key and information identifying the owner, and it is issued and signed by a trusted Certificate Authority (CA).

  1. The client verifies the certificate by checking:
    • The certificate’s digital signature, issued by a trusted Certificate Authority (CA).
    • The domain name in the certificate matches the website’s domain.
    • The certificate’s validity period.
  2. The client generates a temporary session key using the server’s public key and sends it to the server.
  3. The server decrypts the session key using its private key.
  4. All communication between the client and server is encrypted using this session key.

The session key is used because it allows fast encryption of the main data after the secure setup phase is completed.

TLS can also reduce repeated setup work when the same client reconnects to the same server.

  • Session caching is a function in TLS that allows a previous computer session to be remembered, therefore preventing the need to establish a new link each time a new session is attempted.

2.3 Common Applications of SSL/TLS

  • Online banking and shopping (e.g., protecting credit card information).
  • Cloud storage services (e.g., ensuring secure file transfers).
  • Virtual Private Networks (VPNs) for secure remote access.

17.c: Digital Certification

Digital certificates, how they are acquired, and how they are used.

Digital certificates are electronic documents used to verify the identity of individuals, organizations, or websites. They ensure trust and secure communication in online interactions.

3.1 Digital Certificates: Definition and Contents

A digital certificate contains:

  • The owner’s public key.
  • Information about the owner (e.g., domain name, organization name).
  • The CA’s digital signature, which verifies the certificate’s authenticity.
  • Validity dates (start and expiration).

In practice, verification is possible because the CA has a public key that can be used to check the CA’s digital signature on the certificate.

3.2 Certificate Issuance Process

The following image illustrates the main steps involved in obtaining a digital certificate from a Certification Authority (CA).

  1. The owner generates a public-private key pair.
  2. The public key, along with identity information, is submitted to a trusted Certificate Authority.
  3. The CA verifies the owner’s identity and issues a digital certificate.

3.3 Uses of Digital Certificates

3.3.1 Website Authentication

When a client connects to a secure website, the browser verifies the server’s certificate to ensure it is legitimate.

3.3.2 Digital Signatures and Message Integrity

Encryption is mainly used to keep data confidential, while a digital signature is mainly used to prove who sent a message and to prove the message was not changed during transmission.

A digital signature verifies the authenticity and integrity of a message.

The following image shows how a sender creates a digital signature from a message by first producing a digest using a hashing algorithm, then encrypting that digest.

Process
  1. A message is hashed using a hashing algorithm (e.g., SHA-256) to create a digest (a unique numeric summary).
  2. The digest is encrypted with the sender’s private key to form the digital signature.
  3. The recipient decrypts the signature using the sender’s public key and compares it with their own digest of the message. If the digests match, the message is verified as authentic and unaltered.
