16.1a. Maximizing Resource Use in Operating Systems

The OS is responsible for optimizing the use of hardware resources like the CPU, memory, and I/O devices:

CPU Management:

    • Uses scheduling algorithms to allocate processor time efficiently.
      • Example: Round-robin scheduling cycles tasks with equal time slices to prevent any process from monopolizing CPU time‚Äã‚Äã.

Memory Management:

    • Virtual memory combines physical RAM and disk space to handle large applications.
      • Example: Running a 10 GB video editor on a 4 GB RAM machine is enabled by swapping inactive memory pages to disk‚Äã‚Äã.

I/O Optimization:

    • Direct Memory Access (DMA) allows devices like printers to access memory directly without burdening the CPU.
      • Example: Printing documents while the CPU processes other user inputs‚Äã‚Äã.

16.1b. Hiding Hardware Complexities with User Interfaces

Operating systems abstract the complexities of hardware, enabling users to interact easily:

Graphical User Interface (GUI):

    • Offers intuitive, visual interaction with the system.
      • Example: A user copies a file by dragging it into a folder instead of typing a command like cp file.txt /folder‚Äã‚Äã.

Device Drivers:

    • Translate OS commands into device-specific instructions.

16.1c. Process Management

1. Multitasking and Processes

  • A process is a program that is currently being executed by the CPU. It consists of:
    • program code
    • data in memory
    • execution state (registers, program counter, stack)
  • Multitasking allows multiple processes to share the CPU by rapidly switching between them, creating the illusion that they run simultaneously.
  • Multitasking types:
    • Pre-emptive: The operating system can forcibly interrupt a running process and reassign the CPU before the process finishes.
      • Common reasons: the time quantum expires, or a higher-priority process becomes ready.
      • Typical state transition: Running to Ready.
      • Example (time-based): A word processor is paused when its time quantum expires, and the CPU is given to another process.
      • Example (priority-based): A background file download is interrupted when an incoming video call starts, and the CPU is given to the video call.
    • Non-preemptive: A process keeps the CPU until it finishes or requests input/output (I/O), and the operating system does not forcibly interrupt it.
      • Typical state transitions: Running to Waiting, or Running to Terminated.
      • Example (completion): A calculator program runs and finishes its calculation before the CPU is given to another process.
      • Example (blocking): A program requests disk input, moves to the waiting state, and releases the CPU while it waits for the I/O to complete.

2. Process States

  • Process: a program in memory that has an associated process control block
  • Process control block (PCB): a complex data structure containing all data relevant to the running of a process: such as its current state, program counter, CPU registers, and allocated resources, enabling the operating system to manage and transition the process through its states efficiently.
  • Thread: part of a program which is handled as an individual process when being executed

  • Running: Actively executing.
  • Ready: Waiting for CPU allocation.
  • Waiting: Awaiting external events like I/O completion.

3 Scheduling Algorithms

High-level scheduler: makes decisions about which program stored on disk should be moved into memory

Low-level scheduler: makes decisions about which process stored in memory should have access to the CPU:

    • Round Robin:
      • Allocates fixed CPU time slices cyclically to all processes.
        • Example: Three processes share the CPU equally in 10 ms time slots‚Äã‚Äã.
    • Shortest Job First (SJF):
      • Executes tasks with the least execution time first.
    • First Come First Serve (FCFS):
      • Processes are executed in arrival order.
        • Example: Print jobs in a queue‚Äã‚Äã.
    • Shortest Remaining Time (SRT):
      • Dynamically prioritizes tasks with the shortest remaining execution time‚Äã‚Äã.

4 Interrupt Handling and Kernel Role

  • Interrupts: Signals that pause current processes to address high-priority events.
  • Kernel: Manages interrupts, process queues, and resource allocation.
    • Example: A hardware interrupt signals the kernel to handle a disk I/O request‚Äã‚Äã.

16.1d. Memory Management

1. introduction to Virtual Memory, Paging, and Segmentation

  • Virtual Memory:
    • Combines physical memory with disk storage to run larger programs.
    • Example: Running a browser with dozens of tabs open, exceeding physical RAM capacity‚Äã‚Äã
  • Paging:
    • Divides memory into fixed-size blocks (pages) for efficient allocation.
      • Example: A 1 GB program split into pages fits scattered memory slots‚Äã‚Äã.
  • Segmentation:
    • Divides memory into variable-sized segments reflecting logical structures.
      • Example: Separating program code, stack, and data in distinct memory regions‚Äã‚Äã.

2. Paging vs. Segmentation

  • Paging avoids external fragmentation but may waste space within pages.
  • Segmentation aligns memory with program structures but may leave unused gaps between segments‚Äã‚Äã.

3 Page Replacement Algorithms

  • FIFO: Replaces the oldest page in memory.
  • LRU: Removes the least recently accessed page.
    • Example: In a web browser, pages from old sessions are replaced first‚Äã‚Äã.

4 Disk Thrashing

  • Cause: Excessive page swapping between RAM and disk due to insufficient memory.
    • Example: Running too many resource-intensive applications simultaneously degrades performance.
  • Mitigation: Add RAM or reduce the number of active processes‚Äã‚Äã.
