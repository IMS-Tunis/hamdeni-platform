How and Why Computers Use Binary

  • Computers process all data using binary. Each bit is 0 (OFF) or 1 (ON).
  • Logic gates process binary data, and registers are used to temporarily store data.
  • Analogue data must be converted to digital (binary) form to be processed by a computer.
  • The MSB is the leftmost bit; the LSB is the rightmost.
  • Imagine each bit as a tiny light switch: up (1) or down (0)!

Converting Between Number Systems

  • To convert binary to denary: Add the value of each 1 bit (from right: 1, 2, 4, 8...). Example: 1101 = 8 + 4 + 1 = 13.
  • To convert denary to binary: Repeatedly divide by 2 and read remainders from bottom to top.
  • To convert hexadecimal to binary: Replace each hex digit with its 4-bit binary equivalent.
  • To convert binary to hexadecimal: Group binary digits into 4s (right to left), convert each group to hex.
  • To convert denary to hexadecimal: Divide by 16 repeatedly, use remainders (read from bottom to top).
  • To convert hexadecimal to denary: Multiply each digit by its power of 16 and add.
  • Converting is like changing costumes‚Äîsame number, different style!
  • üé≤Example: Convert 23 (denary) to binary: 23 √∑ 2 = 11 r1, 11 √∑ 2 = 5 r1, 5 √∑ 2 = 2 r1, 2 √∑ 2 = 1 r0, 1 √∑ 2 = 0 r1. Read remainders up: 10111.

Why Hexadecimal is Used in Computer Science

  • Hexadecimal is easier for humans to read than binary. Each hex digit represents four binary digits.
  • It's used for error codes, MAC addresses, IPv6, HTML colors, assembly/machine code, and memory dumps.
  • Hex is the computer scientist's secret shorthand!
  • üíªExample: The color code #FF0000 in HTML represents pure red.

Adding Two Positive 8-Bit Binary Integers

  • Binary addition rules: 0+0=0, 1+0=1, 1+1=0 (carry 1), 1+1+1=1 (carry 1). Add from right to left. If the result needs more bits than allowed, overflow occurs.
  • Binary addition: when two bits love each other, they carry over!
  • ‚ûïExample: 0110 + 1011 = 0001 (carry 1 to the next column, etc.).

Concept of Overflow

  • Overflow occurs if a binary sum exceeds the register‚Äôs maximum size (e.g., an extra bit in 8-bit addition). The overflow bit is lost.
  • Overflow is like pouring too much soda in your cup‚Äîwhat spills out is lost!

Logical Binary Shifts

  • Logical left shift: every bit moves left, zero enters on the right (multiplies by 2). Logical right shift: every bit moves right, zero enters on the left (divides by 2). Shifted-out bits are lost.
  • Bits at a dance: step left or right, but if you step off the floor, you‚Äôre out!

Vocabulary (Number Systems)

  • Bit: A single binary digit (0 or 1).
  • Byte: 8 bits.
  • MSB: Most Significant Bit (leftmost).
  • LSB: Least Significant Bit (rightmost).
  • Register: Temporary storage in the CPU.
  • Overflow: Excess bits from a calculation that do not fit.
  • Logic gate: A component processing binary data.
