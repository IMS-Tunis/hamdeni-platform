1. Basic Function Definitions and Usage

  • Declaration: Functions are defined with def followed by the name and parameters in parentheses. The body is indented and may return a value.
  • Calling: Use the function name followed by parentheses; provide arguments inside the parentheses when needed.
Example A - Define a function
def greet(name):
return f"Hello, {name}!"
Example B - Call a function
greeting = greet("Alice")
print(greeting) # Hello, Alice!

2. Parameter Passing and Variable Scope

  • Parameters: Names in the function header that receive argument values when the function is called.
Example B - Use parameters
def multiply(a, b):
return a * b
result = multiply(3, 4)
print(result) # 12
  • Local vs global: Names created inside a function are local. Names defined at top-level are global. Use global to rebind a global name from inside a function (use sparingly).
  • Default and keyword arguments: provide default values and allow naming arguments at the call site.

Reading a global name needs no keyword. Assigning to a global name requires writing global name inside the function.

Example C - Access and modify a global
counter = 0
def increment_counter():
global counter
counter += 1
increment_counter()
print(counter) # 1

Parameter passing

Shows how arguments behave when a function reassigns or mutates them.

Example D - Rebinding vs mutation
def rebind(L):
L = [9]
def mutate(L):
L.append(9)
A = [1, 2]
rebind(A)
mutate(A)
print(A) # [1, 2, 9]
Aliasing: two names, one list
A = [1, 2]
B = A
B.append(99)
print(A) # [1, 2, 99]
Example E - Default and keyword arguments
def power(base, exp=2):
return base ** exp
print(power(5)) # 25
print(power(base=5, exp=3)) # 125

3. Returning Values

  • Single return value: Use return to provide one value to the caller.
Example A - Square a number
def square(x):
return x * x
result = square(5)
print(result) # 25
  • Multiple return values: Return a tuple and optionally unpack it into several variables.
Example B - Return a tuple (min, max)
def get_min_max(numbers):
return min(numbers), max(numbers)
min_val, max_val = get_min_max([1, 2, 3, 4, 5])
print(min_val) # 1
print(max_val) # 5
  • Using returned values: Capture them in variables and reuse later.

If return is omitted, the function returns None.

Example C - Area and perimeter
def calculate_area_and_perimeter(length, width):
area = length * width
perimeter = 2 * (length + width)
return area, perimeter
area, perimeter = calculate_area_and_perimeter(5, 3)
print(f"Area: {area}, Perimeter: {perimeter}")

Multiple return values are returned as one tuple. Unpacking splits the tuple into separate variables.

Returning one value and multiple values (tuple)
Example D - Tuple object and unpacking
def stats(a, b):
return (a + b, a * b)
pair = stats(2, 3)
x, y = pair
print(x, y) # 5 6

4. Functions with Lists

  • Mutation: Lists are mutable; functions can modify them in-place (e.g., append, remove).
Example A - Modify a list in a function
def add_element(my_list):
my_list.append(10)
numbers = [1, 2, 3]
add_element(numbers)
print(numbers) # [1, 2, 3, 10]

To avoid changing the original list, work on a copy: L = L.copy().

  • Returning lists: Functions can construct and return a new list instead of mutating the input.
Example B - Return a new list
def double_elements(my_list):
return [x * 2 for x in my_list]
original = [1, 2, 3]
doubled = double_elements(original)
print(doubled) # [2, 4, 6]

Choose mutation when you intentionally want to change the caller’s list. Choose returning a new list when you want to leave the input unchanged.

Functions with lists: mutate vs return new list
Example C - Work on a copy to avoid mutation
def safe_double(L):
L = L.copy()
return [x * 2 for x in L]
orig = [1, 2, 3]
new_list = safe_double(orig)
print(orig) # [1, 2, 3]
print(new_list) # [2, 4, 6]

5. Using Functions to Implement Algorithms

  • Iteration: Implement loops inside functions to search, count, or transform data.
Example A - Count occurrences
def count_occurrences(item, my_list):
count = 0
for element in my_list:
if element == item:
count += 1
return count
numbers = [1, 2, 3, 2, 4, 2, 5]
print(count_occurrences(2, numbers)) # 3

To stop as soon as a target is found, use return inside the loop. Prefer descriptive names for readability.

  • Conditional logic: Use if, elif, else to return different results based on input values.
Example B - Classify a number
def classify_number(num):
if num > 0:
return "Positive"
elif num < 0:
return "Negative"
else:
return "Zero"
print(classify_number(10)) # Positive
print(classify_number(-5)) # Negative
print(classify_number(0)) # Zero

The function interface (parameters in, return values out) makes algorithms reusable and easy to test.

6. Global Variables and Best Practices

  • global keyword: Declares that a name inside a function refers to a top-level variable, allowing rebinding the global name.
Example A - Increase a global counter
counter = 0
def increase_counter():
global counter
counter += 1
increase_counter()
print(counter) # 1
  • Modify other globals: You can similarly adjust other global counters or accumulators.
Example B - Add to a global total
total = 0
def add_to_total(amount):
global total
total += amount
add_to_total(10)
print(total) # 10
  • Best practices: Use globals sparingly. Prefer passing values as parameters and returning results. Group related state and behavior into functions or classes (encapsulation).

Globals make programs harder to test and understand. If you need shared state, consider passing it explicitly or using a small class to manage it.

Global variables and best practices
Example C - Accumulator without globals
def step(total):
return total + 1
total = 0
total = step(total) # 1
total = step(total) # 2
print(total) # 2
