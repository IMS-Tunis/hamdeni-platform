1. File organization methods:
● Serial File Organization:
o Records are stored one after another in the order they are added, without any
specific order.
o Commonly used for temporary files, such as transaction logs.
o New records are simply appended to the end of the file.
● Sequential File Organization:
o Records are stored in a specific order, typically based on a key field (e.g.,
customer ID).
o New records are inserted in the correct position to maintain the order.
o Efficient for tasks that require processing data in a specific order, such as billing
systems or generating reports.
● Random (or Direct) File Organization:
o Records are stored at positions determined by a hashing algorithm, allowing for
quick retrieval based on key values.
o This method is useful when rapid access to specific records is necessary, such as
in large databases.

File Access Methods:
● Sequential Access:
o Records are accessed in sequence, one by one, starting from the beginning until
the desired record is found.
o Used with serial and sequential files.
o Suitable for tasks that require reading or processing all records, like
end-of-month processing.
● Direct Access:
o Specific records can be accessed directly without reading through other records.
o Applicable to random files and sequential files with an index.
o Ideal for applications requiring quick updates or retrieval of individual records,
such as updating customer information.
Hashing Algorithms:
● Used in random file organization to calculate where records should be stored based on
their key fields.
● A mathematical formula (hash function) is applied to the key to determine the record's
address in the file.
● Hashing algorithms enable fast and efficient data access by minimizing the time needed
to locate records in a file.
Example 1:
if we want to insert the key "42" into a hash table of size 10, the hash function calculates
the index as 42 % 10 = 2, placing the key at index 2. To retrieve the key, we don’t need to
loop through all the table. We can use the same hash function, directing us to the
correct index.
Example 2:
A simple hashing algorithm divides the numeric key field by a chosen number D, with
the remainder determining the storage address in the file. For example, with D=1000
● 0045 / 1000 gives a remainder of 45 (address 45)
● 2005 / 1000 gives a remainder of 5 (address 5)
● 3005 / 1000 also gives a remainder of 5 (address 5)  Collision
● Collisions can happen when two records hash to the same location. This case is managed
through methods like:
● Open Hashing: Finding the next available space.
● Closed Hashing: Using an overflow area to store the colliding record.

Choosing File Organization and Access Methods:
● Serial Files: Best for sequential data processing tasks, such as recording transactions.
● Sequential Files: Ideal when data needs to be accessed or processed in a specific order.
● Random/Direct Access Files: Suitable for large datasets where rapid access to specific
records is needed.
